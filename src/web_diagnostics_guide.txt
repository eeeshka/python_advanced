1) Диагностика DNS-запроса:
 - Команда: dig sfmshop.com
 - Проверяет: есть ли IP-адрес
 - Проблемы: не возвращает ip, ip есть но не открывается,
 - Решение: добавить A-запись в DNS, Проблема в Сервер / сеть / HTTPS

 2) Диагностика TCP-соединения
 - Команда: ping sfmshop.com
 - Проверяем: Есть ли ответ от сервера
 - Проблемы: Сервер не отвечает, обрыв соединения
 - Решение: проверить доступность сервера, проверить сеть

3) Диагностика HTTP-запроса
 - Инструмент: curl -v https://sfmshop.com/products
 - Проверка: отправляется ли запрос, какой статус ответа
 - Проблемы: неправильный запрос
 - Решение: Проверить запрос

4) Диагностика обработки на сервере
 - Инструмент: логи сервера, время ответа
 - Проверка: сколько времени обрабатывается запрос
 - Проблемы: медленные запросы к БД, медленный код
 - Решение: оптимизировать запросы к БД, оптимизировать код

5) Диагностика HTTP-ответа
 - Инструмент: инструменты разработчика браузер
 - Проверка: размер ответа, статус ответа, время передачи
 - Проблемы: большой размер ответа, ошибки в ответе
 - Решение: уменьшить размер ответа, исправить ошибки

6) Диагностика рендеринга
 - Инструмент: инструменты разработчика браузера
 - Проверка: время парсинга, время отрисовки
 - Проблемы: медленный парсинг, сложная отрисовка
 - Решение: оптимизировать данные, упростить отрисовку элементов
