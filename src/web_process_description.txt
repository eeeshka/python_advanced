1) DNS-запрос
 - Это система которая переводит google.com в IP адрес типа 127.00.22.13
 - Первым делом проверяется КЭШ в браузере, может сайт недавно открывался
 - Елси КЭШа нет, отправляется запрос DNS
 - DNS возвращает IP адресс
 - Браузер сохраняет IP адрес в браузере/КЭШе
Оптимальное время запроса 10-50мс
 - диагностика проблем с доступностью сайта, проблема DNS
2) TCP- соединение
браузер отправляет запрос на соединение
Сервер подтверждает
Устанавливается TCP
Теперь можно отправлять/получать данные
Оптимальное время соединения 50-200мс
 - Сервер не отвечает, порт не доступен, проблемы с сетью
3) HTTP-запрос
Отправляется нужный запрос на сервер GET/POST/PUT/DELETE ...
 - Неправильный запрос, некорректная работа API
4)Обработка запроса на сервере
Определяется, что нужно сделать
Делает
Получает ответ
 - проблемы производительности сервера
5) HTTP - ответ
Cервер отправляет HTTP ответ браузеру
оптимальное совместное время для пунктов 3/4/5 составляет 100-500мс
 - диагностика ошибок (статусы 404, 500), работа с api
6) Рендеринг в браузере
браузер получает данные
Отображает данные на странице
Пользователь видит результат
оптимальное время отрисовки страницы 50-200мс
 - диагностика проблем с отображением

Общее время составляет 210-950 мс (0.2-1 сек)
