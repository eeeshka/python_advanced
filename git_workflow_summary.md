# Git Workflow для проекта SFMShop

## Использованные команды Git
- git init - инициализация репозитория
- git add - добавление измененных файлов
- git commit - сохроняет и коментирует изменения
- git status - проверка текущего репозитория, показывает, какие файлы изменены
- git log - просмотр истории комитов
- git branch - создание ветки
- git checkout - переключение на ветку
- git push - отправить изменения на удаленный сервер
- git pull - скачать с удаленного репо

## Созданные ветки
- main - основная ветка кода
- feature/add-discount-system - ветка создания системы скидок
- feature/add-email-validation - ветка создания email валидации
- add-inventory-management - ветка проверки остатков на складе
- feature/add-shipping - ветка расчета стоимости доставки

## Разрешенные конфликты
- конфликт при слиянии в src/models/product.py Разрешен конфликт в get_total_price


## Стратегия работы с ветками
- Ветка `main`: защищена от прямых пушей, изменения попадают только через Pull Request.
- Ветки фич: именуются по шаблону `feature/краткое-описание` (например, `feature/add-discount`).
- Ветки багфиксов: при необходимости можно использовать префикс `fix/`.
- Pull Request: обязателен для любой ветки перед слиянием в `main`. В описании PR указывается, что сделано и как тестировать.
- Ревью: как минимум один разработчик должен просмотреть код и одобрить PR.
- После слияния: ветка удаляется, чтобы не засорять список.
- Актуализация: перед началом работы над новой фичей рекомендуется обновить свою ветку `main` и создать от неё новую ветку.
