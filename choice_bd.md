Для проекта SFMShop лучше подходит Реляционная БД(PostgreSQL, MySQL, SQLite)

1) Имеет четкие связи между данными
Клиент -> Заказ -> товары
2) ссылочная целостность данных
3) если нет пользователя, заказ не создать
4) сложгые запросы, JOIN для полученгия заказов с товарами
5) Струтура данных (Таблицы как раз подходят, не меняется часто)

Когда MongoDB был бы лучше:
1)Кэширование (временное хранение, простые операции)
2) Подходит для быстрых решений, когда структура не определена
3) Логи приложений (быстрое добавление, разный тип данных)
